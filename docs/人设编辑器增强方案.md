# 人设编辑器增强方案

> 目标：让用户能"一整个粘贴复制"塞一套完整人设，而不是逐字段手填。

---

## 现状痛点

| 问题 | 表现 |
|------|------|
| 🔴 逐字段编辑 | `character.yaml` 的每个字段（personality、views、values…）被拆成独立表单控件，想换一套人设得挨个复制 |
| 🔴 没有整体预览 | 看不到人设"拼起来"的完整效果，字段散落在不同位置 |
| 🔴 难以复用 | 想用别人写的角色卡（例如 `.yaml` 格式），没有导入通道 |
| 🔴 切换成本高 | 想换一个性格玩，得手动改十几个字段，还得记着之前的值 |

---

## 方案一：源码模式 — 「塞一个小 VSC」

### 核心思路

在 WebUI 配置页的「角色人设」tab 增加一个 **表单 ↔ 源码** 切换按钮，源码模式下嵌入 **[Monaco Editor](https://microsoft.github.io/monaco-editor/)**（VS Code 的底层编辑器），直接编辑 `character.yaml` 原始内容。

### 架构图

```
┌─────────────────────────────────────────────────┐
│  配置中心                                        │
│  ┌─────┬──────┬──────┬──────┐                    │
│  │ 人设 │ 行为  │ 平台  │ 服务  │  ← Tab 导航     │
│  └─────┴──────┴──────┴──────┘                    │
│                                                 │
│  ┌─ [表单模式] ⇄ [源码模式] ───────────────────┐ │
│  │                                              │ │
│  │  表单模式：                                   │ │
│  │    ChineseName  [___________]                │ │
│  │    personality  [textarea...]                │ │
│  │    values       [textarea...]                │ │
│  │    ...                                       │ │
│  │                                              │ │
│  │  源码模式：                                   │ │
│  │  ┌────────────────────────────────────────┐  │ │
│  │  │ # character.yaml                       │  │ │
│  │  │ character:                             │  │ │
│  │  │   ChineseName: "初念"                  │  │ │
│  │  │   personality:                         │  │ │
│  │  │     - 温柔...                          │  │ │
│  │  │   values:                              │  │ │
│  │  │     - ...                              │  │ │
│  │  │   ... (语法高亮 + 折叠 + 自动补全)      │  │ │
│  │  └────────────────────────────────────────┘  │ │
│  └──────────────────────────────────────────────┘ │
│                                                  │
│          [💾 保存配置]  已保存 ✓                  │
└─────────────────────────────────────────────────┘
```

### 技术方案

| 层面 | 做法 |
|------|------|
| **前端库** | 通过 CDN 引入 `monaco-editor`（约 300KB gzip，按需加载，只在切到源码模式时加载） |
| **数据流** | 表单 ⇄ Monaco 双向同步：切源码时表单数据序列化为 YAML 填入编辑器；切回表单时解析 YAML 回填表单 |
| **后端** | 不变，保存时仍写回 `data/config/character.yaml`，调用 `config_loader.reload()` 刷新 |
| **YAML 校验** | Monaco Editor 的 YAML 语言插件提供实时语法错误提示 |

### 实现步骤

1. 在 `webui/templates/config.html` 的「人设」tab 内，增加源码模式面板和切换按钮
2. 通过 CDN 按需加载 Monaco Editor（`data-onload` 或动态 `<script>`）
3. 实现双向同步逻辑：
   - 表单→源码：用 `js-yaml` 库将表单数据 dump 为 YAML 字符串
   - 源码→表单：用 `js-yaml` 库 parse 回对象，注入到表单控件
4. 后端无需改动（已支持保存 YAML）

### 优点与缺点

| ✅ 优点 | ❌ 缺点 |
|---------|---------|
| 真正的"一整个粘贴复制" | 引入 ~300KB 前端库 |
| 语法高亮、折叠、自动补全 | CDN 需要网络，离线不可用 |
| 用户直接看到 YAML 全貌 | 部分用户可能不熟悉 YAML 语法 |
| 和后端存储格式一致，无转换损耗 | |

## 方案二：人设导入/导出 + 多预设

### 核心思路

不改编辑器形式，在现有表单基础上增加**导出 / 导入 / 预设切换**功能。

### 功能列表

```
┌──────────────────────────────────────────────────┐
│  角色人设                                         │
│                                                  │
│  ┌─ 预设: [初念 ▾] ── [💾 另存为] [🗑 删除] ┐  │
│  └─────────────────────────────────────────────┘  │
│                                                  │
│  [📥 导入 YAML]  [📤 导出 YAML]  [📤 导出 JSON]  │
│                                                  │
│  ┌─ 表单字段 ──────────────────────────────────┐ │
│  │  ChineseName  [初念____________________]    │ │
│  │  personality  [温柔，可爱...            ]   │ │
│  │  ...                                       │ │
│  └─────────────────────────────────────────────┘ │
│                                                  │
│          [💾 保存配置]                            │
└──────────────────────────────────────────────────┘
```

### 具体功能

#### 1️⃣ 导出人设
- 将当前表单所有字段（含嵌套字段）序列化为 YAML 字符串
- 以 `character_初念_2026-05-03.yaml` 文件名下载
- 也提供 JSON 格式导出

#### 2️⃣ 导入人设
- 用户选择本地 `.yaml` 或 `.json` 文件
- 前端 parse 文件内容，校验字段结构
- 自动回填表单所有字段
- **兼容性强**：字段缺失时用默认值填充，多余字段保留在 `extra` 中

#### 3️⃣ 多预设切换
- 在 `data/config/presets/` 目录下按角色名存储多套人设
  ```
  data/config/presets/
    ├── 初念.yaml
    ├── 冷艳御姐.yaml
    └── 呆萌学妹.yaml
  ```
- WebUI 下拉框列出所有预设，点击即可切换
- 支持"另存为预设"和"删除预设"

### 优缺点

| ✅ 优点 | ❌ 缺点 |
|---------|---------|
| 轻量，不引入大型前端库 | 仍然没有"看着 YAML 改"的体验 |
| 纯文件导入导出，兼容性好 | 导入的 YAML 如果格式复杂，可能无法完美映射到扁平表单 |
| 多预设满足日常切换需求 | 用户不能直接在编辑器里看到 YAML 全貌 |

## 方案三：混合方案 ⭐（推荐）

### 核心思路

**方案一 + 方案二**的组合，三合一：

| 功能 | 实现方式 |
|------|----------|
| 🖊️ **源码模式** | Monaco Editor 直接编辑 `character.yaml`，语法高亮 + YAML 校验 |
| 🔄 **双模切换** | 右上角 `表单 ⇄ 源码` 切换，数据双向同步 |
| 📥 **导入** | 支持上传 `.yaml`/`.json` 文件，直接填充源码或表单 |
| 📤 **导出** | 当前人设一键导出为文件 |
| 💾 **多预设** | `data/config/presets/` 目录存储多套角色卡，下拉框切换 |

### 数据流

```
                    ┌──────────────┐
                    │ character.yaml│
                    │  (磁盘文件)   │
                    └──────┬───────┘
                           │
              ┌────────────┴────────────┐
              │  config_loader.reload() │
              └────────────┬────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                                     ▼
┌─────────────────┐              ┌─────────────────────┐
│  表单模式        │              │  源码模式            │
│  (逐字段编辑)    │ ← sync →    │  (Monaco Editor)    │
│                 │              │  YAML 原始编辑       │
└─────────────────┘              └─────────────────────┘
        │                                     │
        └──────────────────┬──────────────────┘
                           ▼
                    ┌──────────────┐
                    │  保存/导出    │
                    └──────────────┘
```

### 预设目录结构

```
data/config/
├── character.yaml          # 当前存档（活动人设）
├── behavior.yaml
├── platforms.yaml
├── routing.yaml
├── services.yaml
└── presets/                # 预设目录
    ├── index.json          # 预设列表索引
    ├── 初念.yaml
    ├── 冷艳御姐.yaml
    └── 呆萌学妹.yaml
```

### 实现步骤

| 步骤 | 内容 | 涉及文件 |
|------|------|----------|
| 1 | 后端新增 `GET /api/config/presets` 列出预设 | `webui/app.py` |
| 2 | 后端新增 `POST /api/config/presets` 保存/删除预设 | `webui/app.py` |
| 3 | 后端新增 `POST /api/config/presets/<name>/apply` 切换预设 | `webui/app.py` |
| 4 | 后端新增 `GET /api/config/export` 和 `POST /api/config/import` | `webui/app.py` |
| 5 | 前端引入 Monaco Editor CDN + js-yaml 库 | `config.html` |
| 6 | 前端实现表单⇄源码双模切换和数据同步 | `config.html` |
| 7 | 前端实现导入导出 UI 和逻辑 | `config.html` |
| 8 | 前端实现预设下拉框和操作按钮 | `config.html` |

### 工作量估算

| 模块 | 预估工时 |
|------|----------|
| 后端 API（预设 + 导入导出） | ~2h |
| 前端 Monaco 集成 + 双模切换 | ~3h |
| 前端导入导出 + 预设 UI | ~2h |
| 联调 + 边界情况处理 | ~1h |
| **总计** | **~8h** |

## 选择建议

| 你的需求 | 推荐方案 |
|----------|----------|
| 就想"一整个粘贴复制"，不要复杂功能 | **方案一** |
| 需要多套人设快速切换，但不想改编辑器 | **方案二** |
| 都要，一步到位 | **方案三 ⭐** |

**我个人推荐方案三**，理由：
1. 源码模式解决"粘贴复制"的核心诉求，是最直观的
2. 预设系统解决了日常切换需求
3. 导入导出让角色卡可以分享、备份
4. 三个功能独立互不干扰，可以分期实现（先做源码模式，再加预设和导入导出）

## 附录：Monaco Editor CDN 集成

```html
<!-- 按需加载，只在源码模式时插入 -->
<script>
function loadMonacoEditor(containerId, yamlContent) {
    return new Promise((resolve) => {
        const loader = document.createElement('script');
        loader.src = 'https://cdn.jsdelivr.net/npm/monaco-editor@0.45.0/min/vs/loader.js';
        loader.onload = () => {
            require.config({ paths: { vs: 'https://cdn.jsdelivr.net/npm/monaco-editor@0.45.0/min/vs' }});
            require(['vs/editor/editor.main'], () => {
                const editor = monaco.editor.create(document.getElementById(containerId), {
                    value: yamlContent,
                    language: 'yaml',
                    theme: 'vs-dark',
                    minimap: { enabled: false },
                    automaticLayout: true,
                    fontSize: 13,
                    lineNumbers: 'on',
                    scrollBeyondLastLine: false,
                    wordWrap: 'on'
                });
                resolve(editor);
            });
        };
        document.head.appendChild(loader);
    });
}
</script>
```

## 附录：YAML 双向同步逻辑

```javascript
// 引入 js-yaml
const yamlLib = window.jsyaml;  // 从 CDN 加载

// 表单 → YAML
function formToYaml(formData) {
    return yamlLib.dump(formData, {
        indent: 2,
        lineWidth: -1,       // 不自动换行
        noRefs: true,
        sortKeys: false,
        forceQuotes: false
    });
}

// YAML → 表单
function yamlToForm(yamlStr) {
    const data = yamlLib.load(yamlStr);
    // 递归填充表单控件
    fillFormFields(data);
    return data;
}
```
