### 3.2 核心代码设计

#### StateManager - 状态管理器

```python
class StateManager:
    """
    管理全局状态和会话级状态
    支持三种响应模式切换
    """

    def __init__(self):
        self.global_mode = ResponseMode.IMMEDIATE
        self.session_modes: Dict[str, ResponseMode] = {}
        self.message_buffer = MessageBuffer()
        self.context_analyzer = ContextAnalyzer()
        self.state_analyzer = StateAnalyzer()  # 异步运行

    def set_mode(self, mode: ResponseMode, session_id: Optional[str] = None):
        """设置响应模式（全局或指定会话）"""
        if session_id:
            self.session_modes[session_id] = mode
        else:
            self.global_mode = mode

    async def handle_message(self, message: ProcessedMessage) -> Optional[str]:
        """
        处理 incoming 消息
        根据当前模式决定如何处理
        """
        mode = self._get_mode_for_message(message)

        if mode == ResponseMode.IMMEDIATE:
            # 立即处理
            return await self._process_immediately(message)

        elif mode == ResponseMode.MENTION_TRIGGER:
            # 提及触发模式
            if self._is_mentioned(message):
                # 被提及，处理缓冲区+当前消息
                buffered = self.message_buffer.get_and_clear(message.group_id)
                return await self._process_batch(buffered + [message])
            else:
                # 未被提及，加入缓冲区
                self.message_buffer.add(message)
                return None

        elif mode == ResponseMode.BUFFER_ALL:
            # 全部缓存
            self.message_buffer.add(message)
            return None

    async def flush_buffer(self, session_id: str):
        """手动触发缓冲区处理（状态变更时调用）"""
        buffered = self.message_buffer.get_and_clear(session_id)
        if buffered:
            return await self._process_batch(buffered)
```

#### StateAnalyzer - 状态分析器（异步）

```python
class StateAnalyzer:
    """
    异步分析会话状态
    检测话题转变、情绪变化、是否需要介入等
    """

    def __init__(self):
        self.client = OpenAI(...)
        self.analysis_queue = asyncio.Queue()
        self._start_worker()

    async def _start_worker(self):
        """启动异步工作线程"""
        while True:
            session_id, messages = await self.analysis_queue.get()
            await self._analyze_async(session_id, messages)

    async def submit_for_analysis(self, session_id: str, messages: List[dict]):
        """提交分析任务"""
        await self.analysis_queue.put((session_id, messages))

    async def _analyze_async(self, session_id: str, messages: List[dict]):
        """
        异步执行状态分析
        结果通过 EventBus 发布，不阻塞主流程
        """
        result = await self._call_llm(messages)

        # 发布状态更新事件
        bus.emit("state_update", {
            "session_id": session_id,
            "topic_shift": result.topic_shift,
            "emotion_change": result.emotion_change,
            "should_intervene": result.should_intervene,
            "suggested_mode": result.suggested_mode,
        })
```

### 3.3 与现有系统的集成点

#### 集成点 1: TaleCore 初始化

```python
class TaleCore:
    def initialize(self):
        # 现有初始化...

        # 【新增】初始化状态管理系统
        self.state_manager = StateManager()
        self._register_state_handlers()

    def _register_state_handlers(self):
        """注册状态相关事件处理器"""
        # 监听状态更新事件（来自异步StateAnalyzer）
        bus.on("state_update", self._handle_state_update)

    def _handle_state_update(self, update: dict):
        """处理状态分析结果"""
        if update.get("should_intervene"):
            # 建议介入，可以自动切换模式或通知用户
            session_id = update["session_id"]
            suggested_mode = update["suggested_mode"]
            self.state_manager.set_mode(suggested_mode, session_id)

            # 如果被建议立即处理，刷新缓冲区
            if suggested_mode == ResponseMode.IMMEDIATE:
                asyncio.create_task(self.state_manager.flush_buffer(session_id))
```

#### 集成点 2: 消息处理流程

```python
class TaleCore:
    async def _process_message_event(self, event_data: dict):
        """处理消息事件"""
        # 转换为 ProcessedMessage
        message = self._convert_event(event_data)

        # 【修改】通过 StateManager 处理消息
        response = await self.state_manager.handle_message(message)

        if response:
            # 获取上下文分析（同步）
            context_info = self.state_manager.context_analyzer.analyze(
                self._get_recent_messages(message.session_id)
            )

            # 构建带上下文的prompt
            enhanced_prompt = self._build_prompt_with_context(
                message, context_info
            )

            # 提交给ChatLLM
            reply = self.chat.chat(enhanced_prompt)

            # 异步提交给StateAnalyzer分析
            await self.state_manager.state_analyzer.submit_for_analysis(
                message.session_id,
                self._get_recent_messages(message.session_id)
            )

            # 发送回复
            await self._send_reply(message, reply)
```

## 4. 工作流程示例

### 场景 1: 群聊提及触发模式

```
用户A: 今天天气怎么样？
用户B: 不知道啊
用户A: @AI 查一下天气

[系统处理]
1. 前两条消息进入缓冲区（未被提及）
2. 检测到@AI，触发处理
3. 批量处理缓冲区+当前消息
4. ChatLLM获得完整上下文，理解用户意图
5. 调用天气工具，回复结果
```

### 场景 2: 异步状态分析触发介入

```
用户A: 我们先聊别的
用户B: 好啊
... (多轮无关对话)

[StateAnalyzer 异步检测]
→ 检测到原话题已结束
→ 发布 state_update 事件
→ 建议切换回 BUFFER 模式

[TaleCore 响应]
→ 自动切换模式
→ 减少不必要的响应
```

## 5. 优势总结

| 特性 | 说明 |
|------|------|
| **解耦** | 状态分析与对话处理分离，互不阻塞 |
| **弹性** | 异步状态分析可容忍高延迟 |
| **可控** | 主ChatLLM保持最终决策权 |
| **灵活** | 支持全局/会话级状态配置 |
| **智能** | 自动检测话题转变，动态调整策略 |

## 6. 实现建议

1. **Phase 1**: 实现 StateManager + MessageBuffer + 三种模式
2. **Phase 2**: 添加 ContextAnalyzer（同步）
3. **Phase 3**: 添加 StateAnalyzer（异步）
4. **Phase 4**: 优化自动状态切换逻辑

这个方案与你们讨论的 "提示而非引导" 设计理念完全一致，主LLM始终保持主导权，辅助LLM只提供额外信息。
